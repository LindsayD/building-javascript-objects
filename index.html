Objects
  • An Object:
  • Represents an instance of a "thing"
  • Stores state (adjectives) about the thing
  • Provides actions (verbs) to interact or change state
  • != a Class

  • Module Pattern Construction Good Stuff:
  • Defines a public API for constructed objects
  • Provides means to specify defaults
  • Makes it easy to create many of the same type
  • Gives ability to keep data/functionality private
  • Module Pattern Construction Bad Stuff:
  • Does not provide "type" checking
  • No automatic instance extension
  • Closure for each object creation may consume more memory than alternatives

  • Prototype Inheritance Good Stuff:
  • Provides runtime augmentation of current and future instances
  • Facilitates composition of data
  • Provides means to specify defaults
  • Makes it easy to create many of the same type
  • Provides a concrete API (and "type" checking)
  • Can saves memory because the prototype defined once
  • Prototype Inheritance Bad Stuff:
  • Object extension can't be used directly on this
  • Prototype extension can cause side effects
  • Not condusive to composition of features
  • Unable to keep data private
  • Cannot protect from API modification at runtime
  • Deep hierachies can get ugly
Which do you use?
  • Use Object Literals
  • As quick data constructs
  • For "throw away" objects
  • Use Object Extension to
  • Combine data with defaults
  • Compose objects from multiple data sources
  • Compose components with Mixins
  • Program in the Functional style
  • Facilitate other forms of object construction
  • Use the Module Pattern to
  • Hide internal implementation
  • Create objects with a steady API when you don't need inheritance
  • Avoid polluting the global scope
  • Implement dependency injection (testing)
  • Implement dependency resolution (dynamic loading)
  • Program in the Functional style
  • Facilitate other forms of object construction
  • Use Prototype Inheritance to
  • Program in the Prototypal style
  • Create an inheritance hierarchy*
  • Feel a bit more like working with classes**
  • Specify default data and behavior
  • Save memory by prototype efficiency
  • Extend objects you might not have created***
  • Implement other forms of object construction
